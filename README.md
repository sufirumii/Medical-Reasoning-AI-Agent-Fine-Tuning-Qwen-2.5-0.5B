# Medical Reasoning AI Agent: Fine-Tuning Qwen-2.5-0.5B

## Project Overview

This repository contains the full pipeline for fine-tuning a Large Language Model (LLM) specifically for clinical reasoning. By utilizing a 500M parameter model and high-quality Chain-of-Thought (CoT) data, this project demonstrates that compact models can achieve structured medical diagnostic capabilities through efficient supervised fine-tuning (SFT).

---

## Model Architecture and Base

| Property | Value |
|---|---|
| Base Model | Qwen-2.5-0.5B-Instruct |
| Parameters | 494 Million |
| Architecture | Transformer-based Causal Decoder |
| Precision | 4-bit NormalFloat (NF4) quantization during training, merged to 16-bit (FP16) for inference |
| Context Window | 2048 Tokens |

---

## Dataset Specifications

The model was fine-tuned on the OpenMed Medical-Reasoning-SFT dataset, derived from the Baichuan-M3-235B model.

| Property | Value |
|---|---|
| Primary Source | OpenMed (Hugging Face) |
| Training Subset | 124,520 samples (initial phase) |
| Extended Target | 1,790,000 samples (full-scale reasoning) |
| Data Format | Conversational JSONL with integrated `reasoning_content` for Chain-of-Thought simulation |

---

## Training Configuration and Metrics

The training was conducted using the Unsloth library to optimize VRAM usage and increase throughput on a single Tesla T4 GPU.

### Hyperparameters

| Parameter | Value |
|---|---|
| Learning Rate | 2e-4 |
| Optimizer | AdamW (8-bit) |
| Batch Size | 1 (per device) |
| Gradient Accumulation | 4 |
| LoRA Rank (R) | 16 |
| LoRA Alpha | 16 |
| Weight Decay | 0.01 |

### Performance Metrics

- Initial Training Loss: 2.3664
- Final Training Loss (1k steps): 1.6457
- Inference Speed: ~2x acceleration via Unsloth Fast Inference kernels

---

## Repository Structure

- `data_transformation.py`: Processes raw medical datasets into the required instruction-response format.
- `fine_tuning.py`: The main training script utilizing LoRA/QLoRA for memory-efficient parameter updates.
- `gradio_agent.py`: A specialized interface for clinical inquiry, featuring a large-text UI for diagnostic review.

---

## Installation and Usage

To replicate this environment, ensure you have a CUDA-compatible GPU and install the following dependencies:

```bash
pip install torch torchvision torchaudio
pip install "unsloth[colab-new] @ git+https://github.com/unslothai/unsloth.git"
pip install gradio
```

### Running the Agent

1. Ensure your merged model is in the directory: `fine-tuning-qwen-openmed/`
2. Execute the Gradio script:

```bash
python gradio_agent.py
```
